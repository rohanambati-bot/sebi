# SentinelSEBI

**SEBI Securities Market TechSprint — Problem Statement 1**
*AI-Driven Detection of Synthetic Media and Phishing Attacks in Securities Markets*

Round 1 submission — working prototype.

---

## 1. Problem → Solution mapping

The problem statement has two halves. SentinelSEBI addresses both with one platform:

| Problem statement gap | Module | What it does |
|---|---|---|
| Hyper-personalised phishing emails, spoofed SMS/WhatsApp, fake regulator/broker messages | **Phishing & Impersonation Scanner** | Scores any message 0–100 for phishing risk with explainable flags (urgency language, credential harvesting, scam-return language, look-alike domains, sender spoofing) |
| AI-generated deepfake images / manipulated visuals of CEOs, CIOs, market experts | **Synthetic Media Forensics** | Runs Error Level Analysis + EXIF forensics on uploaded images, returns a manipulation-risk score and a visual evidence map |
| "Limited mechanisms to verify a communication is genuinely from SEBI / exchange / listed company / intermediary" | **Authenticity Verifier** | Issuers register real communications and get a VERIFY-CODE to publish on the genuine message; investors check a code or paste text to get AUTHENTIC / TAMPERED / UNVERIFIED, including fuzzy-matching against edited copy-paste forwards |

## 2. Target users & channels

- **Retail / first-generation investors** — primary users of the Phishing Scanner and Verifier, reached via a web console (this prototype) and, in the roadmap, a browser/WhatsApp-adjacent extension so they don't have to copy-paste.
- **Brokers & intermediaries** — use the Verifier's issuer side to register outgoing account/KYC/statement communications, reducing their own impersonation exposure.
- **SEBI / exchanges / market infrastructure institutions** — use the Verifier as the source-of-truth registry for official circulars and alerts, and could consume the Phishing Scanner as an API to pre-screen investor complaints.
- **Channels covered**: email, SMS, WhatsApp-style text, and social media post text (Phishing Scanner); images (Media Forensics) — screenshots of "official" posts, executive photos, or forwarded graphics.

## 3. Architecture

```
┌─────────────────────────┐        ┌──────────────────────────────┐
│   Frontend (index.html)  │  REST  │        FastAPI Backend         │
│   vanilla JS + fetch,    │◄──────►│  /phishing/analyze             │
│   no build step needed   │  JSON  │  /verify/register|by-code|...  │
│                          │        │  /media/analyze-image          │
└─────────────────────────┘        └───────────────┬────────────────┘
                                                     │
                                    ┌────────────────┼────────────────┐
                                    ▼                ▼                ▼
                          phishing_engine.py  verify_engine.py  media_engine.py
                          (rule/heuristic      (hash + fuzzy      (Error Level
                           scoring, explainable  content match     Analysis +
                           flags)                registry)         EXIF forensics)
```

Each engine returns a `{risk_score / status, evidence}` shape, so the
heuristics here are drop-in replaceable with trained ML models later without
changing the API contract or frontend (see roadmap below).

## 4. Running it locally

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

Then open `frontend/index.html` directly in a browser (double-click, or
`python3 -m http.server` inside `frontend/` and visit the printed URL). The
API endpoint field at the bottom of the page defaults to
`http://127.0.0.1:8000` — change it if your backend runs elsewhere.

To regenerate the detection-performance evidence:
```bash
cd backend
python3 evaluate.py     # writes docs/EVALUATION.md
```

## 5. Evidence of detection performance

See [`docs/EVALUATION.md`](docs/EVALUATION.md) — generated by `evaluate.py`
against a small hand-labeled sample set (5 phishing / 5 legitimate messages),
reporting accuracy, precision, recall, and F1, plus per-message results. This
is explicitly an MVP-scale sample, not a claim of production-grade benchmark
performance — see the note at the bottom of that file for how we'd scale it.

## 6. What's real vs. roadmap (submitted in good faith)

**Fully functional, running end-to-end in this submission:**
- **Persistent SQLite Database:** Full schema for users, scans, alerts, and fraud reports.
- **PKI Cryptographic verification:** Real RSA-2048 key generation, message private-key signing on registration, and public-key signature verification.
- **Audio Forensic Engine:** Fast Fourier Transform (FFT) frequency roll-off checks, spectral flatness, and digital silence gating to check for voice clones.
- **Video Forensic Engine:** OpenCV face detection, Laplacian spatial blur contrast checking, and temporal facial color histogram stability (flicker checks).
- **QR Certificate Decoder:** Automatic QR code detection and decoding from uploaded image letters using OpenCV.
- **Phishing/impersonation heuristic engine** with fully explainable risk flags.
- **Image forensics:** Genuine Error Level Analysis (ELA) + EXIF metadata checks.
- **Role-based admin console:** SEBI Admins can issue warnings and review reported frauds.

**Round 2 roadmap (out of scope for code-only, offline MVP):**
- Browser/WhatsApp-adjacent extension for one-tap verification.
- Swap heuristic audio/video scorers for trained deep-learning models (like Wave2Vec2 / FaceForensics++ fine-tuned models) behind the same API contracts.
